<a name="seller-insights-definitions"></a>Az értékesítői ismeretek definíciói
=======================

Az alábbi táblázat az értékesítői bepillantást vett számos kifejezés definícióit tartalmazza.

|  **Távú**                 |  **Meghatározás**                                                                                                                              |
|  --------------------------------------------    |  ---------------------------------------------------------------------------------------------------------------------------------           |
| **Azure-licenc típusa**                               | Az ügyfelek által az Azure megvásárlásához használt licencelési szerződés típusa.  Más néven csatorna.                                                  |
| **Azure-licenc típusa: felhőalapú megoldás szolgáltatója**      | A végfelhasználók az Azure-t és a Piactért a felhőalapú megoldás szolgáltatóján keresztül szerzik be, aki viszonteladóként működik.                 |
| **Azure-licenc típusa: Enterprise**                   | A végfelhasználók az Azure-t és a Piactéri ajánlatokat egy Nagyvállalati Szerződéson keresztül szerzik be, közvetlenül a Microsofttal.                  |
| **Azure-licenc típusa: Enterprise a viszonteladón keresztül**  | A végfelhasználók az Azure-t és a Piactéri ajánlatokat egy viszonteladón keresztül szerzik be, amely megkönnyíti a Nagyvállalati Szerződés a Microsofttal.     |
| **Azure-licenc típusa: utólagos fizetés**                | A végfelhasználók az Azure-t és a Piactéri ajánlatokat a "Pay as go" szerződés keretében szerzik be közvetlenül a Microsofttal.                |
| **Díj összege (CC)**                              | A számlázáshoz a vásárlói *pénznemben* (CC) fizetendő összeg.                                                                 |
| **Díj összege (számítógép)**                               | Az ügyfélnek fizetendő összeg a *kifizetési pénznemben* (PC).                                                                      |
| **Díj dátuma**                                      | Az ügyfél költségének kiszámításának dátuma (általában közvetlenül a használati időszak után).                                             |
| **Felhőbeli példány neve**                              | Az a Microsoft Cloud, amelyben a virtuális gép üzembe helyezése megtörtént.                                   |
| **Felhőbeli példány neve: Azure Global**                | A nyilvános globális Microsoft-felhő.                           |
| **Felhőbeli példány neve: Azure Government**        | Kormányzati specifikus Microsoft-felhők a következő kormányok egyikére: Kína, Németország vagy az Amerikai Egyesült Államok Egyesült Államok.                           |
| **Gyűjtemény állapota**  | Egy adott díj esetén a számlázási és a begyűjtési életciklus legújabb állapota.  Előfordulhat például, hogy az állapot gyűjtése folyamatban, összegyűjtve vagy visszatérítésben van.                  |
| **Alapvető méret**  | Az ajánlat által a virtuális gép által bemutatott virtuális magok száma.               |
| **Customer City**  | Az ügyfél által megadott város neve. Ez eltérő lehet az ügyfél Azure-előfizetésében lévő várostól.  |
| **Ügyfél-kommunikációs nyelv**        | Az ügyfél által a kommunikációhoz előnyben részesített nyelv.                                  |
| **Ügyfél vállalatának neve**                  | Az ügyfél által megadott cég neve. Ez eltérő lehet az ügyfél Azure-előfizetésében lévő várostól.                                  |
| **Ügyfél országa**                       | Az ügyfél által megadott ország/régió neve.  Ez eltérő lehet az ügyfél Azure-előfizetésében lévő ország/régió esetében.                               |
| **Ügyfél pénzneme (CC)**                 | Az ügyfél által az árképzéshez és a számlázáshoz előnyben részesített pénznem.                            |
| **Ügyfél e-mail-címe**                         | A végfelhasználó által megadott e-mail-cím.  Ez az ügyfél Azure-előfizetésében szereplő e-mail-címtől eltérő lehet.                   |
| **Ügyfél utóneve**                              | Az ügyfél által megadott keresztnév.  Ez eltérő lehet az ügyfél Azure-előfizetésében megadott névvel.                    |
| **Ügyfél vezetékneve**                               | Az ügyfél által megadott utolsó név.  Ez eltérő lehet az ügyfél Azure-előfizetésében megadott névvel.                    |
| **Ügyfél azonosítója**                               | Az ügyfélhez rendelt egyedi azonosító.  Előfordulhat, hogy egy ügyfél nulla vagy több Azure Marketplace-előfizetéssel rendelkezik.                    |
| **Ügyfél-fizetési típus**                            | Az ügyfél által használt fizetési eszköz típusa.  Ez lehet például kártya, számla vagy elektronikus vizsgálat.                             |
| **Ügyfél postai irányítószáma**                             | Az ügyfél által megadott postai irányítószám.  Ez eltérő lehet az ügyfél Azure-előfizetésében megadott postai kóddal.                                                                 |
| **Ügyfél állapota**                                   | Az ügyfél által megadott állapot (címe).  Ez az ügyfél Azure-előfizetésében megadott vezetéknevtől eltérő lehet.                                                                 |
| **Beszerzés dátuma**                                    | Az első dátum, amikor az ügyfél az Ön által közzétett összes ajánlatot megvásárolta.                 |
| **Elveszett dátum**                                        | Az utolsó időpont, amikor az ügyfél megszakította a korábban megvásárolt összes ajánlatot.                                                         |
| **Becsült kiterjesztett díj (CC)**                   | Az adott SKU (az ügyfél pénznemében) használati egységei mennyiségének becsült kibővített díja. Ez az érték nem lehet pontos a kerekítés vagy a csonkítás hibája miatt.                            |
| **Becsült kiterjesztett díj (PC)**                   | Egy adott SKU használati egységei mennyiségének becsült kibővített díja (a kiadó pénznemében) számított, a deviza-konverzión alapuló, a használatban lévő devizák alapján. Ez az érték nem lehet pontos a kerekítés vagy a csonkítás hibája miatt.                  |
| **Becsült kifizetés (számítógép)**                            | Egy adott SKU használati egységei mennyiségének becsült összege a deviza-konverzió alapján a használat kiszámításának napján (a kiadó pénznemében). Ez az érték nem lehet pontos a kerekítés vagy a csonkítás hibája miatt.                        |
| **Becsült ár (PC)**                             | Egy adott SKU használati egységének becsült árát a dátum kihasználtsága alapján számítjuk ki (a közzétevő pénznemében). Ez az érték nem lehet pontos a kerekítés vagy a csonkítás hibája miatt.                                             |
| **Új ügyfél**                                  | Az érték tudatja Önnel, hogy egy új ügyfél vagy egy vagy több Piactéri ajánlat első alkalommal történő megvásárlása után (vagy nem).  Az érték "yes" lesz, ha a "date beszerzett" dátummal megegyező naptári hónap.  Az érték "nem" lesz, ha az ügyfél a jelentett naptári hónap előtt megvásárolta az ajánlatait.                                                       |
| **Előzetes verziójú SKU**                                  | Az érték tudatja Önnel, ha az SKU-t "Preview"-ként jelölte meg. Az érték "yes" lesz, ha az SKU-t ennek megfelelően címkézték, és csak az Ön által jóváhagyott Azure-előfizetések telepíthetők és használhatók.  Az érték "nem" lesz, ha az SKU nem "Preview"-ként lett azonosítva.                                          |
| **A promóciós kapcsolatfelvételi lehetőség**                   | Az érték tudatja Önnel, hogy az ügyfél proaktív módon választotta-e a közzétevők számára a promóciós kapcsolatfelvételt. Jelenleg nem vesszük igénybe az ügyfeleket, ezért a táblán a "nem" jelölés szerepel. A szolgáltatás üzembe helyezését követően a frissítés ennek megfelelően kezdődik.                                          |
| **Marketplace-licenc típusa**                         | A Piactéri ajánlat számlázási módszere.                             |
| **Marketplace-licenc típusa: számlázás az Azure-ban**   | A Microsoft az Ön ügynöke ehhez a Piactéri ajánlathoz, és az Ön nevében számláz ügyfeleket. (TB hitelkártya vagy vállalati számla)       |
| **Marketplace-licenc típusa: saját licenc használata** | A virtuális gépnek szüksége van az ügyfél által az üzembe helyezéshez biztosított licenckulcs használatára. A Microsoft nem számláz ügyfeleinek ezen piactér-ajánlatok használatáért. |
| **Marketplace-licenc típusa: ingyenes**                   | A Piactéri ajánlat úgy van konfigurálva, hogy az összes felhasználó számára szabadon legyen. A Microsoft nem számlázza az ügyfeleket ennek a Piactéri ajánlatnak a használatára.    |
| **Marketplace-licenc típusa: Microsoft as viszonteladó**  | A Microsoft a piactér-ajánlat viszonteladója.                               |
| **Piactér-előfizetés vállalatának neve**  | Az ügyfél által az Azure-előfizetésében megadott vállalat neve.                           |
| **Piactér-előfizetés azonosítója**            | Az Azure-előfizetéshez társított egyedi azonosító, amelyet az ügyfél a Piactéri ajánlat megvásárlásához használt.  Korábban az Azure-előfizetés GUID-azonosítója volt.  Bizonyos esetekben, például SAAS-sorrend esetén nincs szükség előfizetés megvásárlására.           |
| **Microsoft-díj (CC)**                               | A Microsoft díja a tranzakcióban az ügyfél pénznemében.                              |
| **Ajánlat neve**                                | A Piactéri ajánlat neve.                                                             |
| **Ajánlat típusa**                                | Az Microsoft Marketplace ajánlat típusa.  Például felügyelhető alkalmazás, virtuálisgép-licenc vagy tároló lehet.                           |
| **Megrendelés megszakításának dátuma**                         | A Piactéri rendelés megszakításának dátuma.                                                       |
| **Rendelésazonosító**                                  | A Piactéri szolgáltatáshoz tartozó ügyfél-rendelés egyedi azonosítója.  A virtuális gépek használaton alapuló ajánlatai nincsenek rendeléshez társítva.                 |
| **Megrendelés vásárlásának dátuma**                       | A Piactéri rendelés létrehozásának dátuma.                                                        |
| **Megrendelés mennyisége**                       | Az Azure-előfizetéshez társított azonos SKU-hoz tartozó megrendelések mennyisége.  SAAS-megrendelések esetében az adott megrendelés adott példányához tartozó helyek számát jelöli.                                            |
| **Megrendelés állapota**                              | A Piactéri megrendelés állapota az utolsó frissítés alkalmával.  Például lehet, hogy a sorrend aktív vagy meg lett szakítva.             |
| **Megrendelés állapota: aktív**                           | Az ügyfél megvásárolt egy rendelést, és nem szakította meg a rendelését.   |
| **Megrendelés állapota: megszakítva**                        | Az ügyfél korábban megvásárolt egy rendelést, és később megszakította a rendelését.        |
| **Kifizetési összeg (számítógép)**                             | Az Ön által fizetett összeg az előnyben részesített kifizetési pénznemben (PC).                                       |
| **Kifizetési pénznem (PC)**                           | A kifizetésekhez használt pénznem.                                                                |
| **Kifizetés dátuma**                                    | A Microsofttól a választott pénzügyi intézménynek elküldett fizetési kérelem dátuma.                                                                         |
| **Kifizetés állapota**                                  | A kifizetés állapota az utolsó frissítés időpontjában.                                   |
| **Kifizetés állapota: folyamatban**               | A tranzakció nem áll készen a kifizetésre. (További információ: gyűjtemény állapota)                             |
| **Kifizetés állapota: kifizetve**                          | A tranzakció egy korábbi kifizetési számításban szerepelt. A pozitív értékek kifizetése megtörténik, és a negatív értékek a teljes esedékes összegre vonatkoznak. |
| **Kifizetés állapota: közelgő kifizetés**                   | A tranzakció készen áll a kifizetésre, és szerepelni fog a következő elérhető kifizetési számításban.                      |
| **Ár (CC)**                                   | Egy adott SKU használati egységének díja (az ügyfél pénznemében).                                                                  |
| **Szolgáltatói E-mail cím**                                 | A Microsoft és a végfelhasználó közötti kapcsolatban részt vevő szolgáltató e-mail-címe. Ha az ügyfél a viszonteladón keresztüli vállalat, akkor a viszonteladó.  Ha felhőalapú megoldás-szolgáltató (CSP) van szó, akkor a CSP lesz.                                                                 |
| **Szolgáltató neve**                                 | A Microsoft és a végfelhasználó közötti kapcsolatban részt vevő szolgáltató neve. Ha az ügyfél a viszonteladón keresztüli vállalat, akkor a viszonteladó.  Ha felhőalapú megoldás-szolgáltató (CSP) van szó, akkor a CSP lesz.                                                                 |
| **Erőforrás URI-ja**                                  | Az az elérési út, ahol a telepítési összetevő üzembe lett helyezve.                                            |
| **SKU**                                           | A közzététel során definiált SKU-név. Egy ajánlat több SKU-val is rendelkezhet, de egy SKU csak egyetlen ajánlattal társítható.              |
| **SKU számlázási típusa**                                 | Az SKU számlázási módszere.                                                                                                             |
| **SKU számlázási típusa: saját licenc használata**         | A virtuális géphez vagy szolgáltatáshoz az ügyfél által az üzembe helyezéshez biztosított licenckulcs szükséges. A Microsoft nem számlázza az ügyfeleket ezeknek a Piactéri ajánlatoknak a használatára.   |
| **SKU számlázási típusa: Microsoft as viszonteladóként**          | A Microsoft az Ön viszonteladója ehhez az SKU-hoz.   |
| **SKU számlázási típusa: ingyenes**                           | Az SKU úgy van konfigurálva, hogy az összes felhasználó számára szabadon legyen. A Microsoft nem számlázza az ügyfeleket ennek az SKU-nak a használatára.                           |
| **SKU számlázási típusa: díjköteles**                           | A Microsoft az Ön ügynöke ehhez az SKU-hoz, és az Ön nevében számlázza az ügyfeleket. (TB hitelkártya vagy vállalati számla)                   |
| **SKU számlázási típusa: próbaverzió**                          | Az ügyfél a próbaidőszak alatt van, és akkor kerül kifizetésre, ha nem szakítják meg vagy nem törlik.                                         |
| **Adó összege (CC)**                                  | Az ügyfél számláján az ügyfél pénznemében (CC) alkalmazott áfaösszeg.                                                               |
| **Tranzakció dátuma**                                 | A kifizetési jelentésben rögzített tranzakció dátuma.                                                                           |
| **Tranzakció típusa**                               | A jelentett tranzakció típusa.  Ez lehet például a díj, a visszatérítés vagy a kifizetés beállítása.                                                                                        |
| **Próbaverzió befejezési dátuma**                                 | Az a dátum, ameddig a rendelés próbaidőszaka lejár vagy véget ér.                                                                           |
| **Usage Date (Használat dátuma)**                                       | Az ügyfél használatának dátuma.                                                                                                             |
| **Használat mennyisége**                                       | Az SKU jelentett felhasználói használata. Virtuálisgép-lemezképek esetén a használati rekordok az adott virtuális gép méretének és SKU-jának a jelentett időszakra vonatkozó használatát jelölik.                                               |
| **Használati útmutató**                                       | Egy vagy több napra vonatkozó ügyfél-használat azonosítója a kifizetési jelentésben szereplő bejegyzéshez társított adott SKU esetében.                                               |
| **Usage Type** (Használat típusa)                                       | A mért használat leírása. (Normalizált használat vagy nyers használat)                                                              |
| **Használat típusa: normalizált használat**                | A használati órákat a rendszer a használatban részt vevő virtuálisgép-magok számának (VM-magok száma] x [nyers használati idő]) figyelembevételével normalizálja. A "SHAREDCORE" jelölésű virtuális gépek a 1/6 (vagy 0,1666) értéket használják [virtuálisgép-magok száma] szorzóként.                                        |
| **Használat típusa: nyers használat**                        | A Piactéri ajánlat használati ideje.  A szolgáltatás a "naptár" órája alapján jelenik meg.  Az idő alatt a virtuális gépek száma is fut.                           |
| **Használati egységek**                                   | A megadott felhasználás mértékegysége. A virtuális gépeket mindig óradíjas mértékegységgel mérjük.                               |
| **Virtuális gép mérete**                                       | Az Azure-ajánlattal igazított virtuálisgép-hardver méretét jelöli. Ilyenek `Basic_A0`például `Standard_A11`a `Standard_D12`következők: `Standard_G4`,, és.   |
