# <a name="azure-synapse-analytics-release-notes"></a>Az Azure Synapse Analytics kiadási megjegyzések

Ez a cikk összefoglalja az új funkciók és fejlesztések az [SQL Analytics](sql-data-warehouse-overview-what-is.md) az Azure Synapse legújabb kiadásaiban. A cikk felsorolja azokat a jelentős tartalomfrissítéseket is, amelyek nem kapcsolódnak közvetlenül a kiadáshoz, de ugyanabban az időkeretben teszik közzé. Az Azure-szolgáltatások továbbfejlesztéséről a [Szolgáltatás frissítései](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Az Azure Synapse verziójának ellenőrzése

Mivel az új funkciók minden régióban elérhetők, ellenőrizze a példányra telepített verziót és a legújabb kiadási megjegyzéseket a szolgáltatások elérhetőségéről. A verzió ellenőrzéséhez csatlakozzon az SQL-készlethez az SQL `SELECT @@VERSION;` Server Management Studio (SSMS) segítségével, és futtassa az aktuális verzió visszaadásához.

Az azonosított verzióval ellenőrizze, hogy melyik kiadás lett alkalmazva az SQL-készletre. A kimenetben lévő dátum azonosítja az SQL-készletre alkalmazott kiadás hónapját.

> [!NOTE]
> A SELECT @@VERSION által visszaadott terméknév a Microsoft Azure SQL Data Warehouse-ról az Azure Synapse Analytics szolgáltatásra változik. A módosítás előtt előzetes értesítést küldünk. Ez a változás azon ügyfelek számára fontos, akik@VERSION a SELECT @ alkalmazáskódban szereplő eredménye alapján elemzik a termék nevét. A termék átnevezése miatti alkalmazáskód-módosítások elkerülése érdekében használja ezeket a parancsokat az adatbázis-termék nevéhez és verziójához szükséges KISZOLGÁLÓTULAJDONSÁG lekérdezéséhez: A XX verziószám visszaadozásához. X.XXXXX. X (terméknév nélkül) használja ezt a parancsot: 
> ```
> SELECT SERVERPROPERTY('ProductVersion')
> ```
> Motorkiadás visszaadására használja ezt a parancsot, amely 6 értéket ad vissza az Azure Synapse Analytics (korábban SQL Data Warehouse) számára: 
> ```
> SELECT SERVERPROPERTY('EngineEdition')
> ```

|**A sorszintű biztonság általánosan elérhető**|Az Azure Synapse-ben található SQL Analytics mostantól támogatja a sorszintű biztonság (RLS) használatát, amely hatékony képességgel rendelkezik a bizalmas adatok védelméhez. Az RLS bevezetésével biztonsági házirendeket valósíthat meg a táblák soraihoz való hozzáférés szabályozására, például abban, hogy ki milyen sorokhoz férhet hozzá. Az RLS lehetővé teszi ezt a részletes hozzáférés-vezérlést anélkül, hogy újra kellene terveznie az adatraktárat. Az RLS leegyszerűsíti az általános biztonsági modellt, mivel a hozzáférés-korlátozási logika az adatbázisrétegben található, nem pedig egy másik alkalmazásban lévő adatoktól. Az RLS emellett szükségtelenné teszi a hozzáférés-vezérlési sorok kiszűréséhez szükséges nézetek bevezetését. Ez a nagyvállalati szintű biztonsági funkció nem jár többletköltséggel minden ügyfelünk számára.|
|**Speciális tanácsadók**|Az Azure Synapse-ben az SQL Analytics speciális hangolása egyszerűbbé lett további adattárház-javaslatokkal és mérőszámokkal. Az Azure Advisoron keresztül további speciális teljesítményjavaslatok állnak rendelkezésre, többek között a következők:<br/><br/>1. Adaptív gyorsítótár – Legyen tudni, hogy mikor kell skálázni a gyorsítótár-kihasználtság optimalizálása érdekében.<br/>2. Táblaelosztás – Határozza meg, hogy mikor kell replikálni a táblákat az adatok mozgásának csökkentése és a munkaterhelés teljesítményének növelése érdekében.<br/>3. Tempdb - Megérteni, hogy mikor kell méretezni és konfigurálni az erőforrásosztályokat a tempdb versengés csökkentése érdekében.<br/><br/>Az adattárház-metrikák mélyebb integrációja az [Azure Monitorral,](https://azure.microsoft.com/blog/enhanced-capabilities-to-monitor-manage-and-integrate-sql-data-warehouse-in-the-azure-portal/) beleértve a közel valós idejű metrikák továbbfejlesztett testreszabható figyelési diagramot az áttekintő panelen. A továbbiakban nem kell elhagynia az adattárház áttekintő panel eléréséhez az Azure Monitor metrikák használat figyelése, vagy érvényesítése és alkalmazása adattárház javaslatokat. Emellett vannak új metrikák érhetők el, például a tempdb és az adaptív gyorsítótár-kihasználtság, hogy kiegészítse a teljesítményre vonatkozó javaslatokat.|
|**Fejlett hangolás integrált tanácsadókkal**|Az Azure Synapse speciális hangolása egyszerűbbé lett további adattárház-javaslatokkal és mérőszámokkal, valamint a portál áttekintő paneljének újratervezésével, amely integrált élményt nyújt az Azure Advisorés az Azure Monitor számára.|
|**Gyorsított adatbázis-helyreállítás (ADR)**|Az Azure Synapse accelerated Database Recovery (ADR) most nyilvános előzetes verzióban érhető el. Az ADR egy új SQL Server Engine, amely nagymértékben javítja az adatbázis rendelkezésre állását, különösen hosszú ideig futó tranzakciók esetén, az aktuális helyreállítási folyamat teljes újratervezésével az alapoktól kezdve. Az ADR elsődleges előnyei a gyors és konzisztens adatbázis-helyreállítás és a pillanatnyi tranzakció-visszaállítás.|
|**Az Azure Monitor diagnosztikai naplói**|Az Azure Synapse mostantól lehetővé teszi az analitikus számítási feladatok továbbfejlesztett betekintést az Azure Monitor diagnosztikai naplóival való közvetlen integrálása. Ez az új funkció lehetővé teszi a fejlesztők számára, hogy hosszabb időn keresztül elemezzék a számítási feladatok viselkedését, és megalapozott döntéseket hozzanak a lekérdezésoptimalizálásról vagy a kapacitáskezelésről. Most már bevezetett egy külső naplózási folyamat az [Azure Monitor diagnosztikai naplók,](https://docs.microsoft.com/azure/monitoring/monitoring-data-collection?toc=/azure/azure-monitor/toc.json#logs) amelyek további betekintést nyújt az adattárház számítási feladatok. Egyetlen kattintással konfigurálhatja a diagnosztikai naplókat a korábbi lekérdezési teljesítményhibaelhárítási képességekhez a [Log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-queries)segítségével. Az Azure Monitor diagnosztikai naplói támogatják a testre szabható megőrzési időszakokat azáltal, hogy a naplókat egy tárfiókba menti naplózási célokra, a naplók valós idejű telemetriai adatok közelében történő streamelésére, valamint a naplók elemzésének képességére a Log Analytics használatával naplólekérdezésekkel. A diagnosztikai naplók az adattárház telemetriai nézeteiből állnak, amelyek egyenértékűek az Azure Synapse SQL Analytics-beli dmv-k retasztikájával. Ebben a kezdeti kiadásban engedélyeztük a nézeteket a következő rendszerdinamikus felügyeleti nézetekhez:<br/><br/>&bull; &nbsp; [sys.dm_pdw_exec_requests](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-exec-requests-transact-sql)<br/>&bull; &nbsp; [sys.dm_pdw_request_steps](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-request-steps-transact-sql)<br/>&bull; &nbsp; [sys.dm_pdw_dms_workers](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-dms-workers-transact-sql)<br/>&bull; &nbsp; [sys.dm_pdw_waits](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-waits-transact-sql)<br/>&bull; &nbsp; [sys.dm_pdw_sql_requests](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-sql-requests-transact-sql)|
|**Oszlopcentrikus memória kezelése**|Atömörített oszloptároló sorcsoportok számának növekedésével nő a belső oszlopszegmens metaadatainak kezeléséhez szükséges memória.  Ennek eredményeképpen a lekérdezési teljesítmény és az oszlopcentrikus dinamikus felügyeleti nézetek (DMV-k) egyes lekérdezési teljesítményével és lekérdezésekkel csökkentheti a következőket.  Ebben a kiadásban fejlesztések történtek a belső metaadatok méretének optimalizálása érdekében, ami az ilyen lekérdezések jobb élményéhez és teljesítményéhez vezet.|
|**Azure Data Lake Storage Gen2 integráció (GA)**|A Synapse Analytics mostantól natív integrációt biztosít az Azure Data Lake Storage Gen2 szolgáltatással. Az ügyfelek mostantól külső táblák használatával tölthetnek be adatokat az ABFS-ből az SQL Analytics szolgáltatásba. Ez a funkció lehetővé teszi az ügyfelek számára, hogy integrálódjanak az adattavakkal a Data Lake Storage Gen2-ben.|
