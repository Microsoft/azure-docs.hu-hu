# <a name="security-recommendations---a-reference-guide"></a>Biztonsági ajánlások – útmutató

Ez a cikk felsorolja az Azure Security Centerben esetleg látható javaslatokat. A környezetben megjelenő javaslatok a védeni kívánt erőforrásoktól és a testreszabott konfigurációtól függenek.

Ha többet szeretne tudni arról, hogyan reagálhat ezekre a javaslatokra, olvassa [el a Javaslatok elhárítása az Azure Security Centerben.](security-center-remediate-recommendations.md)

A biztonságos pontszám a Security Center által végrehajtott javaslatok számától függ. Annak eldöntéséhez, hogy mely javaslatokat kell először megoldani, tekintse meg az egyes javaslatok súlyosságát és annak a biztonságos pontszámra gyakorolt lehetséges hatását.

>[!TIP]
> Ha egy javaslat leírása azt mondja: "Nincs kapcsolódó szabályzat", ez általában azért van, mert ez a javaslat egy másik ajánlástól és *annak szabályzatától* függ. Például az "Végpontvédelem állapothibáit ki kell újulni..." ajánlás azon javaslaton alapul, amely ellenőrzi, hogy egy végpontvédelmi megoldás van-e *telepítve* ("Végpontvédelmi megoldást kell telepíteni..."). Az alapul szolgáló *ajánlásnak* van egy irányelve. A házirendek csak az alapajánlásra való korlátozása leegyszerűsíti a házirendek kezelését.
