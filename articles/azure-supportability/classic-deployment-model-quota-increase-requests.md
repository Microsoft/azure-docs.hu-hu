# <a name="classic-deployment-model"></a>Klasszikus üzemi modellben

Klasszikus üzemi modellben az Azure üzembe helyezési mód föld kényszeríti a virtuális gépek és a virtual machine scale sets egy globális vCPU kvótahatárát régebbi generációja. Klasszikus üzemi modell többé nem ajánlott, és a Resource Manager-modell most helyettesít. További információt e két üzembe helyezési modellek és használja ki a Resource Manager tekintse meg Resource Manager üzemi modell lapot. Új előfizetés létrehozásakor a vcpu-k egy alapértelmezett kvóta van hozzárendelve. Visszaállít egy új virtuális Gépet, a klasszikus üzemi modell használatával telepíteni, minden régióban az új és meglévő Vcpu használati összegét nem haladhatja meg a klasszikus üzemi modell esetében hagyja jóvá a vCPU-kvóta. További információ az Azure-előfizetés és a szolgáltatási korlátok lap kvótái.

Kérheti az vcpu-k időkorlátja számára a klasszikus üzemi modellt keresztül súgó + támogatás panelen, vagy a Tanúsítványalgoritmusok + kvóta növelését a portál paneljéről.
