# <a name="get-kubelet-logs-from-azure-kubernetes-service-aks-cluster-nodes"></a>Kubelet-naplók lekérése Azure Kubernetes Service- (AKS-) fürtcsomópontokból

Az AKS-fürt üzemeltetése részeként előfordulhat, hogy a problémák elhárításához át kell tekintenie a naplókat. Az Azure Portal beépített beépített az [AKS-főösszetevők][aks-master-logs] vagy -tárolók naplóinak megtekintésére [egy AKS-fürtben.][azure-container-logs] Esetenként előfordulhat, hogy *kubelet* naplókat kell beszereznie egy AKS-csomópontról hibaelhárítási célokra.

Ez a cikk bemutatja, hogyan tekintheti `journalctl` meg a *kubelet* naplók egy AKS-csomóponton.

## <a name="before-you-begin"></a>Előkészületek

Ez a cikk feltételezi, hogy rendelkezik egy meglévő AKS-fürttel. Ha AKS-fürtre van szüksége, tekintse meg az AKS [gyorsútmutatót az Azure CLI használatával][aks-quickstart-cli] vagy az Azure Portal [használatával.][aks-quickstart-portal]

## <a name="create-an-ssh-connection"></a>SSH-kapcsolat létrehozása

Először hozzon létre egy SSH-kapcsolatot azzal a csomóponttal, amelyen meg kell tekintenie a *kubelet* naplókat. Ez a művelet az [SSH az Azure Kubernetes-szolgáltatás (AKS) fürtcsomópontok][aks-ssh] dokumentum részletesen.

## <a name="get-kubelet-logs"></a>Kubelet-naplók beszerzése

Miután csatlakozott a csomóponthoz, futtassa a következő parancsot a *kubelet* naplók lehívásához:

```console
sudo journalctl -u kubelet -o cat
```
