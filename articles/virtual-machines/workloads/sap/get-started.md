# <a name="use-azure-to-host-and-run-sap-workload-scenarios"></a>SAP-számítási feladatok üzemeltetéséhez és futtatásához használja az Azure-t

A Microsoft Azure használatakor megbízhatóan futtathatja a kritikus fontosságú SAP-számítási feladatokat és forgatókönyveket egy méretezhető, megfelelő és nagyvállalati bevált platformon. Az Azure méretezhetőségét, rugalmasságát és költségmegtakarítását is lehetővé teszi. A Microsoft és az SAP közötti kibővített partnerségnek segítségével sap-alkalmazásokat futtathat fejlesztési és tesztelési és éles forgatókönyvekben az Azure-ban, és teljes mértékben támogatható. Az SAP NetWeavertől az SAP S/4HANA-ig, a Linuxon lévő SAP BI-tól a Windows-ig és az SAP HANA-tól az SQL-ig, lefedjük.

Az SAP NetWeaver-forgatókönyvek üzemeltetése mellett a különböző DBMS az Azure-ban, más SAP számítási feladatok forgatókönyvek, például az Azure-beli SAP BI üzemeltetéséhez. 

Az Azure sap HANA egyedisége egy olyan ajánlat, amely megkülönbözteti az Azure-t. Annak érdekében, hogy több memóriát és CPU-erőforrás-igényes SAP-forgatókönyveket, amelyek magukban foglalják az SAP HANA, az Azure az ügyfél által dedikált operációs rendszer nélküli hardver használatát kínálja. Ezzel a megoldással futtatható az SAP HANA-telepítések, amelyek legfeljebb 24 TB -os (120 TB-os horizontális felskálázás) memóriát igényelaz S/4HANA vagy más SAP HANA számítási feladatok hoz. 

Az SAP-számítási feladatok azure-beli üzemeltetésével az identitásintegráció és az egyszeri bejelentkezés követelményeiis megteremtheti kondi-integrációs követelményeit. Ez a helyzet akkor fordulhat elő, ha az Azure Active Directory (Azure AD) használatával különböző SAP-összetevők és SAP-szoftver-szolgáltatásként (SaaS) vagy platform-as-a-service (PaaS) ajánlatok csatlakoztatása. Az Azure AD- és SAP-entitásokkal való integrációs és egyszeri bejelentkezési forgatókönyvek listáját az "AAD SAP identitásintegráció és egyszeri bejelentkezés" című szakasz ismerteti és dokumentálja.
