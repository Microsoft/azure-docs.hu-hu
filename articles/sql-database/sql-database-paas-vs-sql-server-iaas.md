# <a name="choose-the-right-deployment-option-in-azure-sql"></a>Válassza ki a megfelelő üzembe helyezési lehetőséget az Azure SQL-ben

Ismerje meg, hogy az egyes üzembe helyezési lehetőségek hogyan illeszkednek a Microsoft Azure SQL adatplatformjába, és segítséget kaphat az üzleti igényeinek megfelelő beállításhoz. Akár a költségmegtakarítást, akár a minimális adminisztrációt helyezi előtérbe, ez a cikk segíthet eldönteni, hogy melyik megközelítés felel meg a legfontosabb üzleti követelményeknek.

## <a name="microsofts-azure-sql-data-platform"></a>A Microsoft Azure SQL adatplatformja

A lift-and-shift migrálástól a meglévő alkalmazások korszerűsítéséig, a modern felhőszolgáltatások létrehozásáig az Azure SQL egy modern SQL platform, amely számos telepítési lehetőséget kínál, amelyet az iparág vezető Microsoft SQL Server motorja biztosít. Az Azure SQL-t úgy tervezték, hogy az alkalmazásminták széles skáláját támogassa, amelyek különböző szintű vezérlést biztosítanak az alapul szolgáló platform felett, hogy megfeleljenek a legigényesebb áttelepítési és korszerűsítési követelményeknek. Az Azure SQL egyetlen, egységes felügyeleti élményt nyújtva kiküszöböli az SQL Server-alapú alkalmazások különböző gyűjteményei nagy méretekben történő kezelésének összetettségét.

Az Azure-platformon futó és a helyszíni SQL Server-adatbázisok közötti összehasonlítás első lépése, hogy leszögezzük: semmiről nem kell lemondania. A Microsoft adatplatformja az SQL Server technológiát használja, és elérhetővé teszi azt a fizikai helyszíni gépeken, a magánfelhő-környezetekben, a harmadik fél által üzemeltetett magánfelhő-környezetekben és a nyilvános felhőben. Az SQL Server az Azure virtuális gépeken (SQL virtuális gépek) lehetővé teszi, hogy egyedi és különböző üzleti igényeket elégedjen meg a helyszíni és a felhőben üzemeltetett központi telepítések kombinációjával, miközben ugyanazokat a kiszolgálói termékeket, fejlesztői eszközöket és szakértelmet használja ezeken a környezeteken keresztül.

   ![Felhőalapú SQL Server-verziók: SQL Server infrastruktúra-szolgáltatásban vagy SaaS SQL-adatbázis a felhőben.](./media/sql-database-paas-vs-sql-server-iaas/SQLIAAS_SQL_Server_Cloud_Continuum.png)

Amint az ábrán látható, minden egyes ajánlat az infrastruktúra feletti adminisztrációs szint és a költséghatékonyság mértéke jellemezhető.

Az Azure-ban az SQL Server számítási feladatai üzemeltethetők ([PaaS](https://azure.microsoft.com/overview/what-is-paas/)) vagy üzemeltetett infrastruktúra ([IaaS).](https://azure.microsoft.com/overview/what-is-iaas/) A PaaS-on belül több üzembe helyezési lehetőséggel és szolgáltatási szintekkel rendelkezik az egyes telepítési lehetőségeken belül. A legfontosabb kérdés, amelyet fel kell tennie a PaaS vagy az IaaS közötti döntés során, hogy szeretné-e kezelni az adatbázist, javításokat alkalmazni és biztonsági másolatokat készíteni, vagy delegálja ezeket a műveleteket az Azure-ba?

A választól függően a következő lehetőségek közül választhat:

- [**SQL-adatbázisok**](sql-database-technical-overview.md): A legjobb olyan modern felhőalkalmazások számára, amelyek a legújabb stabil SQL Server-szolgáltatásokat szeretnék használni, és időkorlátokkal rendelkeznek a fejlesztés és a marketing területén. Teljesen felügyelt SQL adatbázis-motor, amely az SQL Server legújabb stabil Enterprise Edition kiadásán alapul. Ez egy relációs adatbázis-szolgáltatásként (DBaaS) az Azure-felhőben, amely a *Platform-as-a-Service (PaaS)* iparági kategóriájába tartozik. Az SQL-adatbázis több telepítési lehetőséggel rendelkezik, amelyek mindegyike a Microsoft tulajdonában lévő, üzemeltetett és karbantartott szabványosított hardverre és szoftverre épül. Az SQL Server használatával olyan beépített funkciókat és funkciókat használhat, amelyek széles körű konfigurálást igényelnek (akár a helyszínen, akár egy Azure virtuális gépen). Az SQL Database használatalapú fizetéssel működik, emellett a teljesítményigény növekedése esetére lehetőség van vertikális vagy horizontális felskálázásra is. Az SQL Database rendelkezik néhány további szolgáltatással, amelyek nem érhetők el az SQL Server ben, például a beépített magas rendelkezésre állás, intelligencia és felügyelet.


  Az adatbázisok a következő telepítési lehetőségeket kínálják:
  - Egyetlen [***adatbázisként,***](sql-database-single-database.md) saját erőforráskészlettel, amelyet egy adatbázis-kiszolgálón keresztül kezelnek. Egyetlen adatbázis hasonló az SQL Server [ben található adatbázishoz.](https://docs.microsoft.com/sql/relational-databases/databases/contained-databases) Ez a lehetőség az új felhőalapú alkalmazások modern alkalmazásfejlesztésére van optimalizálva. [Nagy kapacitású](sql-database-service-tier-hyperscale.md) és [kiszolgáló nélküli](sql-database-serverless.md) lehetőségek állnak rendelkezésre.
  - Egy [***rugalmas készlet***](sql-database-elastic-pool.md), amely adatbázisok gyűjteménye, amelynek megosztott erőforráskészlete egy adatbázis-kiszolgálón keresztül kezelt. Egyetlen adatbázisok áthelyezhetők, és ki egy rugalmas készlet. Ez a beállítás a több-bérlős SaaS-alkalmazásmintával új felhőalapú alkalmazások modern alkalmazásfejlesztésére van optimalizálva. Rugalmas készletek költséghatékony megoldást biztosít a változó használati mintákkal rendelkező adatbázisok teljesítményének kezelésére.
  - [***Adatbázis-kiszolgáló***](sql-database-servers.md), amely az egyes adatbázisok és rugalmas készletek csoportjainak kezelésére szolgál. Az adatbázis-kiszolgálók központi felügyeleti pontként működnek több egy- vagy készletezésű adatbázis, [bejelentkezés,](sql-database-manage-logins.md) [tűzfalszabály,](sql-database-firewall-configure.md) [naplózási szabály](sql-database-auditing.md), [fenyegetésészlelési házirend](sql-database-threat-detection.md)és [feladatátvételi csoport](sql-database-auto-failover-group.md)számára.

- [**SQL által felügyelt példányok**](sql-database-managed-instance.md): A legjobb a legtöbb áttelepítések a felhőbe. A felügyelt példány rendszer- és felhasználói adatbázisok gyűjteménye, amely nek van egy közös erőforráskészlete, amely lift-and-shift készen áll. A legjobb új alkalmazásokhoz vagy meglévő helyszíni alkalmazásokhoz, amelyek a legújabb stabil SQL Server-szolgáltatásokat szeretnék használni, és amelyek minimális módosításokkal kerülnek át a felhőbe. A felügyelt példány hasonló a [Microsoft SQL Server adatbázis-kezelő motor](https://docs.microsoft.com/sql/database-engine/sql-server-database-engine-overview) egy példányához, amely megosztott erőforrásokat kínál adatbázisokhoz és további példányhatókörű szolgáltatásokhoz. A felügyelt példány támogatja az adatbázis-áttelepítést a helyszíni adatbázisból, minimális vagy egyetlen adatbázis-módosítás nélkül. Ez a beállítás az Azure SQL Database összes PaaS-előnyét biztosítja, de olyan képességeket ad hozzá, amelyek korábban csak az SQL virtuális gépeken érhetők el. Ez magában foglalja a natív virtuális hálózat (VNet) és közel 100%-os kompatibilitása a helyszíni SQL Server. A felügyelt példányok teljes SQL Server-hozzáférést és szolgáltatáskompatibilitást biztosítanak az SQL-kiszolgálók Azure-ba való áttelepítéséhez.


- [**SQL virtuális gépek**](../virtual-machines/windows/sql/virtual-machines-windows-sql-server-iaas-overview.md): Az operációs rendszer szintű hozzáférést igénylő áttelepítésekhez és alkalmazásokhoz ajánlott. Az SQL virtuális gépek lift-and-shift készen állnak a meglévő alkalmazások, amelyek megkövetelik a gyors áttelepítés a felhőbe minimális módosításokkal, vagy nincs változás. Az SQL virtuális gépek teljes körű felügyeleti vezérlést biztosítanak az SQL Server-példány és az Azure-ba való migrálás alapjául szolgáló operációs rendszer felett. Ha gyorsan kell elvégezni a fejlesztést és a tesztelést, és nem szeretne helyszíni nem üzemi SQL Server-hardvert vásárolni. Az SQL virtuális gépek az *Infrastruktúra szolgáltatásként (IaaS)* iparági kategóriába tartoznak, és lehetővé teszik az SQL Server futtatását egy teljesen felügyelt virtuális gépen (VM) belül az Azure-felhőben. Az SQL virtuális gépek a Microsoft tulajdonában lévő, üzemeltetett és karbantartott szabványosított hardveren is futnak. SQL virtuális gépek használata esetén vagy használatalapú fizetés az SQL Server-lemezképben már szereplő SQL Server licencért, vagy egyszerűen használhat egy meglévő licencet. A virtuális gép szükség szerint leissza vagy folytathatja. A felhőben telepített és üzemeltetett SQL Server az Azure-on futó Windows Server vagy Linux virtuális gépeken fut, más néven infrastruktúra szolgáltatásként (IaaS). Az SQL virtuális gépek jó lehetőséget kínálnak a helyszíni SQL Server-adatbázisok és -alkalmazások adatbázis-módosítás nélküli áttelepítésére. Az SQL Server összes legújabb verziója és kiadása telepíthető egy IaaS virtuális gépen. A legjelentősebb különbség az SQL-adatbázisok és az SQL felügyelt példányok, hogy az SQL Server virtuális gépek lehetővé teszik a teljes körű vezérlést az adatbázis-motor. Kiválaszthatja, hogy mikor szeretné elindítani a karbantartást/javítást, módosíthatja a helyreállítási modellt egyszerű vagy tömegesen naplózott ra, szükség esetén szüneteltetheti vagy elindíthatja a szolgáltatást, és teljes mértékben testreszabhatja az SQL Server adatbázis-motorját. Ezzel a további vezérlő jön a hozzáadott felelőssége a virtuális gép kezelésére.

  Meglévő alkalmazások Azure-ba való áttelepítésére vagy meglévő helyszíni alkalmazások felhőre való kiterjesztésére van optimalizálva hibrid környezetekben. Emellett a virtuális gépen futó SQL Servert használhatja hagyományos SQL Server-alkalmazások fejlesztésére és tesztelésére is. Az SQL virtuális gépek, rendelkezik a teljes felügyeleti jogokkal egy dedikált SQL Server-példány és egy felhőalapú virtuális gép felett. Ez tökéletes választás, ha a vállalatnál adva vannak a virtuális gépek fenntartásához szükséges számítástechnikai erőforrások. E lehetőségeknek köszönhetően a legapróbb részletekig testre szabhatja a rendszert, úgy, hogy az a leginkább megfeleljen az alkalmazás teljesítmény- és rendelkezésre állási követelményeinek.


A további különbségek az alábbi táblázatban találhatók, de mind az adatbázisok, mind a ***felügyelt példányok úgy vannak optimalizálva, hogy az általános felügyeleti költségeket a lehető legkisebbre csökkentsék számos adatbázis kiépítéséhez és kezeléséhez.*** Ez a megoldás csökkenti az adminisztrációra fordítandó összeget, mivel használata esetén nincs szükség virtuális gépek, operációs rendszerek vagy adatbázisszoftverek kezelésére. Nem kell foglalkoznia a frissítések, a magas rendelkezésre állás és a [biztonsági mentések](sql-database-automated-backups.md) kezelésével sem. Összefoglalva elmondható, hogy az Azure SQL Database segítségével egyetlen számítástechnikai vagy fejlesztői erőforrás jóval nagyobb számú adatbázis kezelésére válik alkalmassá. [Rugalmas készletek](sql-database-elastic-pool.md) is támogatja a SaaS több-bérlős alkalmazásarchitektúrák funkciókkal, beleértve a bérlők elkülönítése és a költségek csökkentése az erőforrások megosztása az adatbázisok között. [A felügyelt példány](sql-database-managed-instance.md) támogatja a példányható körrel ellátott szolgáltatásokat, amelyek lehetővé teszik a meglévő alkalmazások egyszerű áttelepítését, valamint az erőforrások adatbázisok közötti megosztását.

## <a name="business-motivations-for-choosing-databases-managed-instances-or-sql-virtual-machines"></a>Az adatbázisok, felügyelt példányok vagy SQL-virtuális gépek kiválasztásának üzleti motivációi

Számos tényező befolyásolhatja a különböző adatajánlatok közötti választásra vonatkozó döntését:

- [Költség](#cost) – Mind a PaaS, mind az IaaS opció tartalmazza az alapárat, amely lefedi az alapul szolgáló infrastruktúrát és a licencelést. Az IaaS-beállítással azonban további időt és erőforrásokat kell befektetnie az adatbázis kezeléséhez, míg a PaaS-ban ezeket a felügyeleti funkciókat az ár tartalmazza. IaaS opció lehetővé teszi, hogy állítsa le az erőforrásokat, miközben nem használja őket a költségek csökkentésére, míg a PaaS-verzió mindig fut, kivéve, ha csepp, és újra létrehozza az erőforrásokat, amikor szükség van rájuk.
- [Felügyelet](#administration) – A PaaS-beállítások csökkentik az adatbázis felügyeletéhez szükséges időt. Azonban azt is korlátozza az egyéni felügyeleti feladatok és parancsfájlok, amelyek végrehajtható vagy futtatható. Például a CLR nem támogatott egy- vagy készletezésű adatbázisok, de támogatja a felügyelt példány. Emellett a PaaS egyetlen telepítési lehetősége sem támogatja a nyomkövetési jelzők használatát.
- [Szolgáltatásiszintű szerződés](#service-level-agreement-sla) – Az IaaS és a PaaS egyaránt magas, iparági szabványnak megfelelő SLA-t biztosít. A PaaS opció 99,99%-os SLA-t garantál, míg az IaaS 99,95%-os SLA-t garantál az infrastruktúrához, ami azt jelenti, hogy további mechanizmusokat kell bevezetnie az adatbázisok rendelkezésre állásának biztosítása érdekében. A magas rendelkezésre állású megoldás megvalósítása 99,99%-os, hozzon létre egy további SQL Server virtuális gép, és konfigurálja AlwaysOn rendelkezésre állási csoportok.
- [Ideje az Azure-ba való áthelyezés](#market) – Az Sql Server az Azure VM-ben a környezet pontos egyezése, így a helyszíni AZURE-beli virtuális gépről való migrálás nem különbözik attól, mint az adatbázisok áthelyezése az egyik helyszíni kiszolgálóról a másikra. A felügyelt példány rendkívül egyszerű áttelepítést tesz lehetővé; előfordulhat azonban, hogy a felügyelt példányra való áttelepítés előtt alkalmaznia kell néhány módosítást.

Ezeket a tényezőket a következő szakaszokban részletesebben tárgyaljuk.

### <a name="cost"></a>Költségek

Az adatbázisok üzemeltetési módjának kiválasztását általában a rendelkezésre álló források határozzák meg, és ez egyformán igaz, akár szűkös költségvetésű startupról, akár egy nagyvállalat szigorú költségkeretek közé szorított osztályáról van szó. Ebben a szakaszban megismerheti az Azure számlázási és licencelési alapjait a két relációs adatbázis-beállítással kapcsolatban: az SQL Database és az SQL virtuális gépek. Megismerheti az alkalmazás összköltségének kiszámításának módját is.

#### <a name="billing-and-licensing-basics"></a>A számlázás és a licencelés alapjai

Dinamikusan módosíthatja a szolgáltatási szinteket és a számítási méreteket, hogy megfeleljen az alkalmazás változatos átviteli igényeinek.

Az **SQL-adatbázisok és**az SQL felügyelt példányok segítségével az adatbázis-szoftvert a Microsoft automatikusan konfigurálja, kijavítja és frissíti, ami csökkenti a felügyeleti költségeket. Ezenfelül a [beépített biztonsági mentési](sql-database-automated-backups.md) funkciókkal is jelentős költségmegtakarítást érhet el, különösen, ha nagyszámú adatbázist használ.

### <a name="administration"></a>Adminisztráció

Számos vállalat számára a felhőszolgáltatásokra történő átállást az összetett adminisztrációs feladatok kiszervezése ösztönzi, és nem csupán a költségszempontok. Az IaaS és a PaaS segítségével a Microsoft felügyeli az alapul szolgáló infrastruktúrát, és automatikusan replikálja az összes adatot a vészhelyreállítás biztosítása érdekében, konfigurálja és frissíti az adatbázisszoftvert, kezeli a terheléselosztást, és átlátszó feladatátvételt végez, ha van egy kiszolgáló meghibásodása egy adatközponton belül.

- Az **SQL-adatbázisok és az SQL által felügyelt példányok**segítségével folytathatja az adatbázis felügyeletét, de már nem kell kezelnie az adatbázismotort, az operációs rendszert vagy a hardvert. Ön felelős például a következő területekért: adatbázisok és bejelentkezési adatok, indexek és lekérdezések hangolása, naplózás és biztonság. Emellett a magas rendelkezésre állás konfigurálása egy másik adatközpontban minimális konfigurációt és felügyeletet igényel.
- Az **SQL virtuális gépek**segítségével teljes mértékben szabályozhatja az operációs rendszer és az SQL Server példánykonfigurációját. A virtuális gépen Ön dönti el, hogy mikor frissíti az operációs rendszert és az adatbázisszoftvert, illetve, hogy telepít-e kiegészítő programokat, például víruskereső vagy biztonsági mentési eszközöket. Egyes automatizált szolgáltatások jelentősen leegyszerűsítik a frissítést, a biztonsági mentések készítését és a magas rendelkezésre állást. Ezenfelül meghatározhatja a virtuális gép méretét, a lemezek számát, valamint a tárhely konfigurációját is. Az Azure-ban igény szerint módosítható a virtuális gépek mérete. További információk: [Virtual Machine and Cloud Service Sizes for Azure](../virtual-machines/windows/sizes.md) (Virtuális gépek és felhőszolgáltatások mérete az Azure-ban).

### <a name="service-level-agreement-sla"></a>Szolgáltatói szerződés (SLA)

Számos számítástechnikai osztály számára elsődleges prioritást jelent a szolgáltatói szerződésben (SLA) vállalt üzemidő biztosítása. Ebben a részben áttekintjük, hogy milyen  SLA vonatkozik az egyes adatbázis-üzemeltetési megoldásokra.

Az **SQL Database**esetében a Microsoft 99,99%-os rendelkezésre állási SLA-t biztosít. További információk: [SLA a következőhöz: SQL-adatbázis](https://azure.microsoft.com/support/legal/sla/sql-database/).

Az **SQL virtuális gépek**, a Microsoft 99,95%-os rendelkezésre állási SLA-t biztosít, amely csak a virtuális gépet fedi le. Ez az SLA nem érvényes a virtuális gépen futó folyamatokra (például az SQL Serverre), ezenfelül legalább két, rendelkezésre állási csoportba rendezett virtuálisgép-példány futtatása is szükséges hozzá. A legfrissebb információkért lásd: [SLA a következőhöz: Virtuális gépek](https://azure.microsoft.com/support/legal/sla/virtual-machines/). A virtuális gépeken belüli magas rendelkezésre állású adatbázis (HA) esetén konfigurálnia kell az SQL Server egyik támogatott magas rendelkezésre állási beállítását, például [az Always On Availability Groups (Mindig rendelkezésre állási csoportokon) beállítást.](https://docs.microsoft.com/sql/database-engine/availability-groups/windows/always-on-availability-groups-sql-server) A támogatott magas rendelkezésre állási lehetőségek valamelyikének használata nem biztosít további SLA-t, de lehetővé teszi az adatbázis 99,99% fölötti rendelkezésre állásának elérését.

### <a name="time-to-move-to-azure"></a><a name="market"></a>Ideje az Azure-ra költözni

**Az SQL-adatbázis** (egyetlen adatbázis vagy rugalmas készlet) a megfelelő megoldás a felhőtervezésű alkalmazásokhoz, ha a fejlesztői hatékonyság és az új megoldások gyors piacra dobása kritikus fontosságú. A DBA-hoz hasonló programozási funkciói révén tökéletes választás a felhőben dolgozó tervezők és fejlesztők számára, mivel csökkenti az alkalmazás mögötti operációs rendszer és adatbázis felügyeletére fordítandó időt.

**Az SQL által felügyelt példány** nagymértékben leegyszerűsíti a meglévő alkalmazások Azure SQL-be való migrálását, lehetővé téve az áttelepített adatbázis-alkalmazások gyors piacra vietrálását az Azure-ban.

**Az SQL virtuális gépek** tökéletesek, ha a meglévő vagy új alkalmazások nagy adatbázisokat vagy az SQL Server vagy a Windows/Linux összes szolgáltatásához való hozzáférést igényelnek, és el szeretné kerülni az új helyszíni hardver beszerzésének idejét és költségeit. Ez is egy jó illeszkedést, ha azt szeretné, hogy a meglévő helyszíni alkalmazások és adatbázisok azure-as-is - azokban az esetekben, ahol az Azure SQL Database felügyelt példány nem megfelelő. Mivel nem kell módosítania a bemutatót, az alkalmazást és az adatrétegeket, időt és költségvetést takaríthat meg a meglévő megoldás újramegtervezéséhez. Ehelyett termékeinek az Azure-ra történő áttelepítésére, illetve az Azure-platformon való működéshez esetleg szükséges teljesítményoptimalizálásra összpontosíthat. További információk: [Performance Best Practices for SQL Server on Azure Virtual Machines](../virtual-machines/windows/sql/virtual-machines-windows-sql-performance.md) (Teljesítményre vonatkozó ajánlott eljárások az Azure Virtual Machines szolgáltatásban futtatott SQL Server esetében).
