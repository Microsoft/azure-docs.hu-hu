# <a name="azure-cognitive-services-support-and-feedback-options"></a>Azure Cognitive Services támogatás és visszajelzési lehetőségek

Most megkezdi az Azure Cognitive Services funkcióinak megismerését? Lehet, hogy új funkciót implementál az alkalmazásban. Vagy a szolgáltatás használata után javaslatai vannak a fejlesztéséhez? Itt találhat olyan lehetőségeket, amelyekkel támogatást kaphat, naprakész maradhat, visszajelzéseket küldhet, és hibákat jelenthet a Cognitive Services.

## <a name="get-support"></a>Támogatás kérése

### <a name="create-an-azure-support-request"></a>Azure-támogatáskérés létrehozása

<div class='icon is-large'>
    <img alt='Azure support' src='https://docs.microsoft.com/media/logos/logo_azure.svg'>
</div>

Fedezze fel az [Azure-támogatási lehetőségek körét, és válassza ki a](https://azure.microsoft.com/support/plans) legjobban illeszkedő csomagot, legyen szó akár fejlesztőknek, akár a Felhőbeli utazásról, akár egy olyan nagy szervezetről, amely üzleti szempontból kritikus fontosságú, stratégiai alkalmazásokat helyez üzembe. Az Azure-ügyfelek támogatási kérelmeket hozhatnak létre és kezelhetnek a Azure Portalban.

* [Azure Portal](https://ms.portal.azure.com/#blade/Microsoft_Azure_Support/HelpAndSupportBlade/overview)
* [A Egyesült Államok Government Azure Portal](https://portal.azure.us)

### <a name="search"></a>Keresés

<div class='icon is-large'>
    <img alt='Search' src='https://docs.microsoft.com/media/common/i_search.svg'>
</div>

A gyorsabb eredmény érdekében keressen olyan releváns webhelyekre, mint a [stack overflow](https://stackoverflow.com/), a [Microsoft docs](https://docs.microsoft.com/azure/cognitive-services/)vagy a [GitHub-kód mintája](https://github.com/azure-samples). Használja a `site:` lekérdezési jelölést a [kedvenc keresőmotorjában](https://bing.com), például:

```
{search keywords} site:stackoverflow.com
```

Hol `{search keywords}` található a keresés kontextusa. Érdemes lehet ezeket a hatókörön belüli kereséseket megfelelően használni:

 - Stack Overflow:`site:stackoverflow.com`
 - Microsoft Docs:`site:docs.microsoft.com`
 - GitHub-minták:`site:github.com/azure-samples`

### <a name="post-a-question-on-stack-overflow"></a>Kérdés közzététele Stack Overflow

<div class='icon is-large'>
    <img alt='Stack Overflow' src='https://docs.microsoft.com/media/logos/logo_stackoverflow.svg'>
</div>

Ha nem talál választ a problémára a Stack Overflow keresési mezőjében, küldjön be egy új kérdést. Stack Overflow a fejlesztéssel kapcsolatos kérdések előnyben részesített csatornája. A Stack Overflow Közösség és a Microsoft csapattagok tagjai közvetlenül részt vesznek a problémák megoldásában.

Tegye [fel kérdéseit](https://go.microsoft.com/fwlink/?linkid=2126464). Használja a megfelelő címkét a kérdéséhez, így biztosak lehetünk abban, hogy megtekintjük a kérdést.

> [!TIP]
> A Stack Overflow következő bejegyzései a kérdések és a forráskód hozzáadására vonatkozó tippeket tartalmaznak. Az irányelvek követésével növelheti az esélyeket, hogy a Közösség tagjai gyorsan felmérjék és reagálnak a kérdéseire:
> * [Hogyan kérdezzen egy jó kérdésre?](https://stackoverflow.com/help/how-to-ask)
> * [Hogyan hozható létre egy minimális, reprodukálható példa?](https://stackoverflow.com/help/minimal-reproducible-example)

## <a name="report-bugs"></a>Hibák jelentése

### <a name="create-a-github-issue-or-pull-request"></a>GitHub-probléma vagy pull-kérelem létrehozása

<div class='icon is-large'>
    <img alt='GitHub' src='https://docs.microsoft.com/media/logos/logo_octokitty.svg'>
</div>

Az alábbiakban háromféle tárházat talál, ahol a fejlesztő hibát tehet közzé vagy lekéréses kérelmet hozhat létre. Probléma létrehozásához használja a megfelelő GitHub-tárház *problémák* lapját, és válassza az *új probléma*lehetőséget. Ha szeretne kijavítani egy javítást, küldje el a lekéréses kérelmet közvetlenül a fájl szerkesztésével. A lekéréses kérelem ezután a *pull-kérelmek* lapon fog megjelenni, amíg az adattár tulajdonosai meg nem nézhetik.
