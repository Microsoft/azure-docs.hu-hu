# <a name="azure-sql-data-warehouse-release-notes"></a>Azure SQL Data Warehouse kibocsátási megjegyzések

Ez a cikk a [Azure SQL Data Warehouse](sql-data-warehouse-overview-what-is.md) (Azure SQL DW) legújabb kiadásaiban található új szolgáltatásokat és fejlesztéseket foglalja össze. A cikk a kiadáshoz közvetlenül kapcsolódó, de ugyanabban az időkeretben közzétett, jelentős tartalmi frissítéseket is felsorolja. Az egyéb Azure-szolgáltatásokkal kapcsolatos további információkért lásd: [szolgáltatások frissítései](https://azure.microsoft.com/updates).

## <a name="check-your-azure-sql-data-warehouse-version"></a>A Azure SQL Data Warehouse verziójának keresése

Mivel az új funkciók minden régióban elérhetők, tekintse meg a példányra telepített verziót, valamint a legújabb Azure SQL DW kibocsátási megjegyzéseket a szolgáltatás rendelkezésre állásához. Az Azure SQL DW verziójának vizsgálatához kapcsolódjon az adattárházhoz SQL Server Management Studio (SSMS) keresztül, `SELECT @@VERSION;` és futtassa az parancsot az Azure SQL DW aktuális verziójának visszaadásához.

Példa a kimenetre:

![SQL Data Warehouse verziója](./media/release-notes/sql_data_warehouse_version.png)

Az azonosított dátum használatával erősítse meg, hogy melyik kiadás lett alkalmazva az Azure SQL DW-re.

A VNet szolgáltatás végpontjai lehetővé teszik, hogy elkülönítse a logikai kiszolgálóhoz való kapcsolódást egy adott alhálózatról vagy alhálózatokból a virtuális hálózaton belül. Azure Data Lake Storage Gen2 minden olyan tulajdonságot bevezet, amely az elemzési és az Azure Storage-ba való teljes életciklusához szükséges. A két meglévő Azure Storage-szolgáltatás, az Azure Blob Storage és a Azure Data Lake Storage Gen1 funkcióit konvergáljuk. A [Azure Data Lake Storage Gen1](https://docs.microsoft.com/azure/data-lake-store/index), például a fájlrendszer szemantikai, a fájl szintű biztonság és a skálázás funkciói az [Azure Blob Storage](https://docs.microsoft.com/azure/storage/blobs/storage-blobs-introduction)alacsony díjas, többszintű tárolással és magas rendelkezésre állású/vész-helyreállítási funkcióival kombinálhatók.<br/><br/>A Base használatával a VNet-be védett Azure Storage-ba is importálhat adatok Azure SQL Data Warehouseba. Hasonlóképpen, a VNet-ből a Azure SQL Data Warehouseról az Azure Storage-be védett adatok exportálását is támogatja a rendszer a Base-on keresztül.<br/><br/>A Azure SQL Data Warehouse VNet szolgáltatásbeli végpontokkal kapcsolatos további információkért tekintse meg a [blogbejegyzésben](https://azure.microsoft.com/blog/general-availability-of-vnet-service-endpoints-for-azure-sql-data-warehouse/) vagy a [dokumentációban](https://docs.microsoft.com/azure/sql-database/sql-database-vnet-service-endpoint-rule-overview?toc=/azure/sql-data-warehouse/toc.json).|
|**Automatikus Teljesítményfigyelés (előzetes verzió)**|A [query Store](https://docs.microsoft.com/sql/relational-databases/performance/monitoring-performance-by-using-the-query-store?view=sql-server-2017) már előzetes verzióban érhető el a Azure SQL Data Warehouse. A lekérdezési tároló engedélyezése olyan egyszerű, mint az ALTER DATABASE T-SQL-utasítás futtatása: <br/>SQL----ALTER DATABASE [DatabaseName] SET QUERY_STORE = ON;-------a lekérdezési Azure SQL Data Warehouse tárolóval kapcsolatos további információért tekintse meg a következő cikket: a [teljesítmény figyelése a lekérdezési tároló használatával](/sql/relational-databases/performance/monitoring-performance-by-using-the-query-store)és a lekérdezés-áruház DMV, például a [sys. query_store_ lekérdezés](/sql/relational-databases/system-catalog-views/sys-query-store-query-transact-sql). |**Alacsonyabb számítási szintek Azure SQL Data Warehouse Gen2**|Azure SQL Data Warehouse Gen2 mostantól támogatja az alacsonyabb számítási szinteket. Az Azure SQL Data Warehouse ügyfelek a 100 cDWU ([adattárház-egységek](what-is-a-data-warehouse-unit-dwu-cdwu.md)) kezdődő teljesítmény-, rugalmassági és biztonsági funkciókkal, valamint percek alatt méretezhetők a 30 000 cDWU. A 2018-től kezdődően az ügyfelek a Gen2 teljesítményének és rugalmasságának köszönhetően kihasználhatják a [régiókban](gen2-migration-schedule.md#automated-schedule-and-region-availability-table)lévő alacsonyabb számítási szinteket, és a 2019-as időszakban elérhető további régiókat is igénybe vehetik.<br/><br/>Ha elhagyják a belépési pontot a következő generációs adattárházak számára, a Microsoft megnyitja az ajtót olyan értékkel rendelkező ügyfeleknek, akik egy biztonságos, nagy teljesítményű adattárház összes előnyét szeretnék kipróbálni, és nem sejtette, hogy melyik próbaverziós környezet a legmegfelelőbb számukra. Az ügyfelek az aktuális 500 cDWU beléptetési ponttól kezdve akár 100 cDWU is elindíthatók. SQL Data Warehouse a Gen2 továbbra is támogatja a szüneteltetést, és folytatja a műveleteket, és nem csupán a számítási rugalmasságot illeti. A Gen2 Emellett támogatja a korlátlan számú oszlopos tárolási kapacitást, valamint a 2,5-szor több memóriát a lekérdezésekben, akár 128 egyidejű lekérdezéseket és [adaptív gyorsítótárazási](https://azure.microsoft.com/blog/adaptive-caching-powers-azure-sql-data-warehouse-performance-gains/) funkciókat. |**Oszlopcentrikus háttérbeli egyesítése**|Alapértelmezés szerint a Azure SQL Data Warehouse (Azure SQL DW) oszlopos formátumban tárolja az értékeket, a [sorcsoportokba való tömörítéséhez](sql-data-warehouse-memory-optimizations-for-columnstore-compression.md)nevű mikro-partíciókkal. Időnként, az index Build vagy az betöltési idő miatti memória-korlátozás miatt előfordulhat, hogy a sorcsoportokba való tömörítéséhez a 1 000 000-es sorok optimális méretéhez képest kevesebbet tömörít. A sorcsoportokba való tömörítéséhez a törlések miatt is töredezett lehet. A kis-és a töredezett sorcsoportokba való tömörítéséhez nagyobb memória-használatot, valamint nem hatékony lekérdezés-végrehajtást eredményeznek. Az Azure SQL DW ezen kiadásával a oszlopcentrikus háttér-karbantartási feladata összefésüli a kis tömörített sorcsoportokba való tömörítéséhez, hogy nagyobb sorcsoportokba való tömörítéséhez hozzon létre a memória jobb kihasználásához és a lekérdezések végrehajtásának felgyorsításához.
