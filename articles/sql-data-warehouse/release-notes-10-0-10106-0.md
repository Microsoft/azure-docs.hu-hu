# <a name="azure-synapse-analytics-release-notes"></a>Az Azure szinapszis Analytics kibocsátási megjegyzései

Ez a cikk összefoglalja az Azure Szinapszisban az [SQL Analytics](sql-data-warehouse-overview-what-is.md) legújabb verzióinak új funkcióit és fejlesztéseit. A cikk a kiadáshoz közvetlenül kapcsolódó, de ugyanabban az időkeretben közzétett, jelentős tartalmi frissítéseket is felsorolja. Az egyéb Azure-szolgáltatásokkal kapcsolatos további információkért lásd: [szolgáltatások frissítései](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Az Azure szinapszis verziójának megtekintése

Mivel az új funkciók minden régióban elérhetők, tekintse meg a példányra telepített verziót, valamint a szolgáltatás rendelkezésre állásának legújabb kibocsátási megjegyzéseit. A verzió vizsgálatához kapcsolódjon az SQL-készlethez SQL Server Management Studioon (SSMS) keresztül, és futtassa `SELECT @@VERSION;` az aktuális verzió visszaadásához.

Az azonosított verzió segítségével ellenőrizze, hogy az SQL-készleten melyik kiadás lett alkalmazva.

> [!NOTE]
> Június 2020-án a @@VERSION által visszaadott terméknév a Microsoft Azure SQL Data Warehouseról az Azure szinapszis Analyticsre változik. A kibocsátási megjegyzésekben közzé fogjuk tenni az ütemtervet. Ez a változás olyan ügyfelek esetében fontos, akik a terméknév elemzését az alkalmazás kódjában válassza a @@VERSION. Ha el szeretné kerülni az alkalmazás kódjának módosítását a termék újrahasznosítása miatt, a következő parancsokkal lekérdezheti a SERVERPROPERTY az adatbázis-terméknév és-verzió számára: a XX-es verziószámot adja vissza. X. XXXXX. X (Terméknév nélkül) használja ezt a parancsot: 
> ```
> SELECT SERVERPROPERTY('ProductVersion')
> ```
> A motor kiadásának visszaküldéséhez használja ezt a parancsot, amely az Azure szinapszis Analytics 6 verzióját adja vissza (korábban SQL Data Warehouse): 
> ```
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Ehhez a nagyvállalati szintű biztonsági szolgáltatáshoz nem tartozik további díj az összes ügyfelünk számára.|
|**Speciális tanácsadók**|Az SQL Analytics speciális finomhangolása az Azure Szinapszisban egyszerűen elérhető további adattárház-javaslatokkal és mérőszámokkal. További speciális teljesítményre vonatkozó javaslatok érhetők el Azure Advisor az Ön rendelkezésére áll, beleértve a következőket:<br/><br/>1. adaptív gyorsítótár – ajánlott a gyorsítótár kihasználtságának optimalizálása a méretezéshez.<br/>2. tábla eloszlása – meghatározza, hogy mikor replikálja a táblákat az adatáthelyezés csökkentése és a munkaterhelés teljesítményének növeléséhez.<br/>3. tempdb – Ismerje meg, hogy az erőforrás-osztályok méretezése és konfigurálása hogyan csökkenthető a tempdb-tartalom csökkentése érdekében.<br/><br/>Az adatraktár metrikáinak mélyebb integrációja [Azure monitor](https://azure.microsoft.com/blog/enhanced-capabilities-to-monitor-manage-and-integrate-sql-data-warehouse-in-the-azure-portal/) , beleértve a közel valós idejű metrikák kibővített testreszabható figyelési diagramját az Áttekintés panelen. A használat figyelése, illetve az adatraktár-javaslatok érvényesítése és alkalmazása során az adatraktár áttekintő paneljén már nem kell elhagynia Azure Monitor metrikák elérését. Emellett új mérőszámok érhetők el, például a tempdb és az adaptív gyorsítótár kihasználtsága a teljesítményre vonatkozó javaslatok kiegészítéseként.|
|**Speciális hangolás integrált tanácsadókkal**|Az Azure szinapszis speciális finomhangolása egyszerűen elérhető további adattárház-javaslatokkal és-mérőszámokkal, valamint a portál áttekintés paneljének újratervezésével, amely integrált élményt nyújt Azure Advisor és Azure Monitor.|
|**Gyorsított adatbázis-helyreállítás (ADR)**|Az Azure szinapszis gyorsított adatbázis-helyreállítás (ADR) már nyilvános előzetes verzióban érhető el. Az ADR egy új SQL Server motor, amely nagy mértékben javítja az adatbázisok rendelkezésre állását, különösen a hosszú ideig futó tranzakciók jelenlétében, az alapoktól kezdve egészen a jelenlegi helyreállítási folyamat teljes újratervezésével. Az ADR elsődleges előnyei a gyors és konzisztens adatbázis-helyreállítás és a tranzakciók azonnali visszaállítása.|
|**Diagnosztikai naplók Azure Monitor**|Az Azure szinapszis mostantól lehetővé teszi az analitikai számítási feladatok továbbfejlesztett elemzését azáltal, hogy közvetlenül Azure Monitor diagnosztikai naplókkal integrálja őket. Ez az új funkció lehetővé teszi, hogy a fejlesztők a számítási feladatok viselkedését hosszabb időn keresztül elemezzék, és megalapozott döntéseket hozhassanak a lekérdezés optimalizálása vagy a kapacitások kezelése terén. Bevezetünk egy külső naplózási folyamatot [Azure monitor diagnosztikai naplókon](https://docs.microsoft.com/azure/monitoring/monitoring-data-collection?toc=/azure/azure-monitor/toc.json#logs) keresztül, amelyek további elemzéseket biztosítanak az adattárház munkaterheléséről. A gombra kattintva egyetlen kattintással beállíthatja a diagnosztikai naplókat a [log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-queries)használatával végzett korábbi lekérdezések teljesítményének hibaelhárítási képességeihez. Azure Monitor a diagnosztikai naplók lehetővé teszik a testreszabható megőrzési időszakok használatát azáltal, hogy a naplókat egy Storage-fiókba mentik a naplózási célokra, így a naplók a valós idejű telemetria-elemzések közelében továbbítják a naplókat, és képesek elemezni a naplókat a Log Analytics használatával a log lekérdezésekkel. A diagnosztikai naplók az adatraktár telemetria-nézeteiből állnak, amelyek az Azure Szinapszisban az SQL-elemzések leggyakrabban használt teljesítmény-hibaelhárítási DMV felelnek meg. Ennél a kezdeti kiadásnál engedélyezve vannak a következő rendszerdinamikus felügyeleti nézetek nézetei:<br/><br/>&bull; &nbsp; [sys. dm_pdw_exec_requests](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-exec-requests-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_request_steps](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-request-steps-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_dms_workers](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-dms-workers-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_waits](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-waits-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_sql_requests](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-sql-requests-transact-sql)|
|**Oszlopcentrikus memória kezelése**|Mivel a tömörített oszlopdiagram száma nő, a sorcsoportokba való tömörítéséhez-szegmensek belső oszlopaihoz tartozó metaadatok kezeléséhez szükséges memória növekszik.  Ennek eredményeképpen a lekérdezési teljesítmény és a Oszlopcentrikus dinamikus felügyeleti nézetei (DMV) által végrehajtott lekérdezések csökkenhetnek.  Ebben a kiadásban fejlesztéseket hajtottak végre a belső metaadatok méretének optimalizálása érdekében ezekben az esetekben, ami jobb élményt és teljesítményt nyújt az ilyen lekérdezésekhez.|
|**Azure Data Lake Storage Gen2 integráció (GA)**|A szinapszis Analytics mostantól natív módon integrálható Azure Data Lake Storage Gen2okkal. Az ügyfelek mostantól a ABFS külső tábláival is betölthetik az adatait az SQL Analytics szolgáltatásba. Ez a funkció lehetővé teszi, hogy az ügyfelek integrálva legyenek Data Lake Storage Gen2ban található adattavakkal.|
