# <a name="azure-sql-data-warehouse-release-notes"></a>Azure SQL Data Warehouse kibocsátási megjegyzések

Ez a cikk a [Azure SQL Data Warehouse](sql-data-warehouse-overview-what-is.md) (Azure SQL DW) legújabb kiadásaiban található új szolgáltatásokat és fejlesztéseket foglalja össze. A cikk a kiadáshoz közvetlenül kapcsolódó, de ugyanabban az időkeretben közzétett, jelentős tartalmi frissítéseket is felsorolja. Az egyéb Azure-szolgáltatásokkal kapcsolatos további információkért lásd: [szolgáltatások frissítései](https://azure.microsoft.com/updates).

## <a name="check-your-azure-sql-data-warehouse-version"></a>A Azure SQL Data Warehouse verziójának keresése

Mivel az új funkciók minden régióban elérhetők, tekintse meg a példányra telepített verziót, valamint a legújabb Azure SQL DW kibocsátási megjegyzéseket a szolgáltatás rendelkezésre állásához. Az Azure SQL DW verziójának megadásához kapcsolódjon az adattárházhoz SQL Server Management Studio (SSMS) keresztül, és futtassa `SELECT @@VERSION;` az Azure SQL DW aktuális verziójának visszaadásához.

Példa a kimenetre:

![SQL Data Warehouse verziója](./media/release-notes/t47-1-version.png)

Az azonosított verzió használatával erősítse meg, hogy melyik kiadás lett alkalmazva az Azure SQL DW-re.

Ez lehetővé teszi, hogy a későbbi lekérdezés-végrehajtások közvetlenül a megőrzött gyorsítótárból kapjanak eredményeket, így az újraszámításra nincs szükség. Az eredményhalmaz gyorsítótárazása javítja a lekérdezési teljesítményt, és csökkenti a számítási erőforrások használatát. Emellett a gyorsítótárazott eredményeket használó lekérdezések nem használnak párhuzamossági tárolóhelyeket, így nem számítanak bele a meglévő egyidejűségi korlátokba. A biztonság érdekében a felhasználók csak akkor érhetik el a gyorsítótárazott eredményeket, ha ugyanazokat az adatelérési engedélyeket használják, mint a gyorsítótárazott eredményeket létrehozó felhasználók. További információ: [a teljesítmény finomhangolása az eredményhalmaz gyorsítótárazásával](/azure/sql-data-warehouse/performance-tuning-result-set-caching). |**Speciális tanácsadók**|A Azure SQL Data Warehouse (SQL DW) speciális finomhangolása egyszerűen megtalálhatók a további adattárház-javaslatokkal és mérőszámokkal. További speciális teljesítményre vonatkozó javaslatok érhetők el Azure Advisor az Ön rendelkezésére áll, beleértve a következőket:<br/><br/>1. adaptív gyorsítótár – ajánlott a gyorsítótár kihasználtságának optimalizálása a méretezéshez.<br/>2. tábla eloszlása – meghatározza, hogy mikor replikálja a táblákat az adatáthelyezés csökkentése és a munkaterhelés teljesítményének növeléséhez.<br/>3. tempdb – Ismerje meg, hogy az erőforrás-osztályok méretezése és konfigurálása hogyan csökkenthető a tempdb-tartalom csökkentése érdekében.<br/><br/>Az adatraktár metrikáinak mélyebb integrációja [Azure monitor](https://azure.microsoft.com/blog/enhanced-capabilities-to-monitor-manage-and-integrate-sql-data-warehouse-in-the-azure-portal/) , beleértve a közel valós idejű metrikák kibővített testreszabható figyelési diagramját az Áttekintés panelen. Ezentúl nem kell elhagynia az adattárház áttekintő paneljét ahhoz, hogy lássa az Azure Monitor metrikáit a használat monitorozásakor, illetve az adattárházra vonatkozó ajánlások ellenőrzésekor és alkalmazásakor. Emellett új mérőszámok érhetők el, például a tempdb és az adaptív gyorsítótár kihasználtsága a teljesítményre vonatkozó javaslatok kiegészítéseként.|

|**Gyorsított adatbázis-helyreállítás (ADR)**|Azure SQL Data Warehouse gyorsított adatbázis-helyreállítás (ADR) már nyilvános előzetes verzióban érhető el. Az ADR egy új SQL Server motor, amely nagy mértékben javítja az adatbázisok rendelkezésre állását, különösen a hosszú ideig futó tranzakciók jelenlétében, az alapoktól kezdve egészen a jelenlegi helyreállítási folyamat teljes újratervezésével. Az ADR elsődleges előnyei a gyors és konzisztens adatbázis-helyreállítás és a tranzakciók azonnali visszaállítása.|
|**Diagnosztikai naplók Azure Monitor**|A SQL Data Warehouse (SQL DW) mostantól lehetővé teszi az analitikai számítási feladatokhoz való fokozott betekintést azáltal, hogy közvetlenül a Azure Monitor diagnosztikai naplókat integrálja. Ez az új funkció lehetővé teszi, hogy a fejlesztők a számítási feladatok viselkedését hosszabb időn keresztül elemezzék, és megalapozott döntéseket hozhassanak a lekérdezés optimalizálása vagy a kapacitások kezelése terén. Bevezetünk egy külső naplózási folyamatot [Azure monitor diagnosztikai naplókon](https://docs.microsoft.com/azure/monitoring/monitoring-data-collection?toc=/azure/azure-monitor/toc.json#logs) keresztül, amelyek további elemzéseket biztosítanak az adattárház munkaterheléséről. A gombra kattintva egyetlen kattintással beállíthatja a diagnosztikai naplókat a [log Analytics](https://docs.microsoft.com/azure/log-analytics/log-analytics-queries)használatával végzett korábbi lekérdezések teljesítményének hibaelhárítási képességeihez. Azure Monitor a diagnosztikai naplók lehetővé teszik a testreszabható megőrzési időszakok használatát azáltal, hogy a naplókat egy Storage-fiókba mentik a naplózási célokra, így a naplók a valós idejű telemetria-elemzések közelében továbbítják a naplókat, és képesek elemezni a naplókat a Log Analytics használatával a log lekérdezésekkel. A diagnosztikai naplók az adattárház olyan telemetriai nézeteiből állnak, amelyek egyenértékűek az SQL Data Warehouse teljesítményproblémáinak elhárításához leggyakrabban használt DMV lekérdezésekkel. Ennél a kezdeti kiadásnál engedélyezve vannak a következő rendszerdinamikus felügyeleti nézetek nézetei:<br/><br/>&bull; &nbsp; [sys. dm_pdw_exec_requests](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-exec-requests-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_request_steps](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-request-steps-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_dms_workers](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-dms-workers-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_waits](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-waits-transact-sql)<br/>&bull; &nbsp; [sys. dm_pdw_sql_requests](/sql/relational-databases/system-dynamic-management-views/sys-dm-pdw-sql-requests-transact-sql)|
|**Oszlopcentrikus memória kezelése**|Mivel a tömörített oszlopdiagram száma nő, a sorcsoportokba való tömörítéséhez-szegmensek belső oszlopaihoz tartozó metaadatok kezeléséhez szükséges memória növekszik.  Ennek eredményeképpen a lekérdezési teljesítmény és a Oszlopcentrikus dinamikus felügyeleti nézetei (DMV) által végrehajtott lekérdezések csökkenhetnek.  Ebben a kiadásban fejlesztéseket hajtottak végre a belső metaadatok méretének optimalizálása érdekében ezekben az esetekben, ami jobb élményt és teljesítményt nyújt az ilyen lekérdezésekhez.|
